## 参与 Nodeloc Wiki 的贡献

Nodeloc Wiki 欢迎社区用户参与内容完善和更新。您可以通过提交 **Pull Request (PR)** 的方式来提供修改、补充或纠正信息，但请务必遵守以下规则：

### 提交 PR 的要求

1. **明确修改目的**  
   - 在 PR 标题和描述中清晰说明修改的原因。  
   - 示例：
     - “修正论坛发帖操作步骤错误”  
     - “补充积分获取规则说明”  
     - “更新常见问题中已过期的链接”

2. **遵守内容规范**  
   - 避免发布广告、敏感信息或不相关内容。  
   - 保持语言简明、准确、尊重社区文化。  
   - 尽量提供引用或来源，以便验证信息。

3. **分支和提交规范**  
   - 在本地创建新分支进行修改，避免直接在 main 分支修改。  
   - 提交时尽量拆分为小步提交，每个提交只处理一类修改。

4. **审查流程**  
   - 所有 PR 都会由管理员或维护者审核。  
   - 通过审核后，修改才会合并到主分支并发布。

### 参考示例

```text
PR 标题：修正“私信通知设置”步骤
PR 描述：
- 修复原文步骤遗漏“保存设置”操作
- 补充通知偏好截图示例
- 原因：新用户反馈按照原文操作无法生效
```
